Exercise 1 Consider these four storybook functions and their characteristic shapes:

  • flat(): Output doesn’t change regardless of input
  • steady(): Output increases at a constant rate as input increases
  • hillside(): Output starts low, gradually increases, then levels off at a higher value
  • hill(): Output rises from a baseline to a peak, then falls back to the baseline

For each real-world scenario below, choose which storybook function would best model the relationship between the input and output.

Exercise 4 This exercise focuses on input and output transformations—the technique for adapting storybook functions to model real-world data with actual units and dimensions.

The setup: You’re modeling the spread of a viral video. The data shows: - At week 0, essentially no one had seen it (views ≈ 0) - By week 10, about half of the eventual maximum viewers had seen it - By week 20, views had essentially plateaued at 2 million

The hillside() function naturally: - Outputs ≈ 0 for large negative inputs - Outputs 0.5 when input = 0 - Outputs ≈ 1 for large positive inputs

Part A: Understanding the input transformation

The input transformation has the form \(a(q - q_0)\) where \(q\) is the real-world quantity (weeks, in this case).

  1. We want hillside() to output 0.5 when the week is 10. What value should \(q_0\) have?
rpg-q0
0       0.5       5       10       20      

  1. We want hillside() to be near its maximum (output ≈ 1) around week 20. The input to hillside() should be around +2 when week = 20. Given that \(q_0 = 10\), solve for \(a\) in: \(a(20 - 10) = 2\)
rpg-a
0.1       0.2       0.5       1       2      

  1. What are the units of parameter \(a\)?
rpg-a-units
weeks       per week       views       dimensionless      

Part B: Understanding the output transformation

The output transformation has the form \(A \cdot \text{hillside}(\ldots) + B\).

  1. The hillside() function naturally ranges from 0 to 1. We want our model to range from 0 to 2,000,000 views. What should \(A\) be?
rpg-A
1       1000       1e+06       2e+06      

  1. What are the units of parameter \(A\)?
rpg-A-units
weeks       per week       views       dimensionless      

  1. Since hillside() naturally starts at 0 (for very negative inputs) and we want our model to also start at 0 views, what should \(B\) be?
rpg-B
-1e+06       0       1e+06       2e+06      

Part C: The complete model

  1. Putting it all together, which expression represents the complete model for views as a function of week?
rpg-complete

hillside(0.2 * (week - 10))

2000000 * hillside(0.2 * (week - 10))

2000000 * hillside(week - 10)

hillside(2000000 * (week - 10))

  1. According to this model, approximately how many views would there be at week 5?
rpg-week5
about 150,000       about 500,000       about 1,000,000       about 1,500,000
